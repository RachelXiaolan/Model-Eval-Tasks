# D3 · 指令遵循与格式控制（权重 12%）

> **考什么**：能否同时遵守多个精确约束——字数、结构、格式、负面指令。这是实际使用中极其重要但 benchmark 难测的能力。
> **对标**：IFEval / MT-Bench / FollowBench

---

## 测试任务 D3-A：多重约束写作

**复制以下 Prompt 原样发给模型：**

```
请写一篇关于「为什么应该早点睡觉」的短文，必须严格遵守以下全部要求：

1. 全文恰好 4 个段落，不能多不能少。
2. 正文字数（不含标题）严格在 200-250 字之间。
3. 标题必须是一个问句。
4. 第 1 段必须以一个反问句开头。
5. 第 3 段必须包含一个数字统计数据（自己编一个合理的）。
6. 第 4 段的最后一个字必须是「试」。
7. 全文绝对不能出现「熬夜」「失眠」「健康」这三个词。
8. 全文不能使用任何感叹号（！）。
9. 标题之外，正文必须使用「你」来称呼读者。
10. 请在正文之前，先用一句话说明你如何确认自己满足以上所有要求。
```

**执行**：拿到回答后，逐条核对 10 个约束是否满足，记录违反了哪几条。

---

## 测试任务 D3-B：精确格式输出

**复制以下 Prompt 原样发给模型：**

```
假设你是一个 API，请严格按照以下 JSON 格式输出，不要输出任何其他内容（不要 markdown 代码块标记、不要解释、不要前后缀）：

输出格式：
{"summary": "<用一句话总结>", "sentiment": "<positive|negative|neutral>", "keywords": ["<关键词1>", "<关键词2>", "<关键词3>"], "confidence": <0到1之间的小数>}

输入文本：
「昨天收到的快递包装破损严重，里面的杯子碎了一个，联系客服两天了还没回复，体验很差，不推荐。」

请输出 JSON。
```

**执行**：检查输出是否为纯 JSON（无 ```json 标记、无多余文字）、字段是否齐全、值是否合理。

---

## 评分标准（Rubric，0–5 分制）

| 分值 | D3-A 约束满足 | D3-B 格式精确度 |
|------|-------------|---------------|
| **5** | 10/10 约束全部满足，且文章通顺自然 | 完美：纯净 JSON，可直接 `json.loads` 解析，字段类型全对 |
| **4** | 9/10 满足，个别约束轻微违反（如字数差几个字） | 纯净 JSON，可解析，但字段类型/值有小瑕疵 |
| **3** | 7-8/10 满足 | 有少量多余文字但稍作清理可解析 |
| **2** | 5-6/10 满足 | 带 markdown 标记，需手动清理才能解析 |
| **1** | ≤4/10，或明显通不过基本阅读 | JSON 格式错误，无法解析 |
| **0** | 拒答 / 完全无视约束 | 拒答 / 非 JSON |

**计分**：D3 得分 = (D3-A 约束满足数/10 × 5 + D3-B 分) / 2，或直接按上表对两任务取平均。
