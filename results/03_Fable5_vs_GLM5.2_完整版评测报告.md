#### 🚨 GLM5.2 愤怒小鸟严重 bug（测试人实跑发现）

测试人在本地浏览器实跑 GLM5.2 生成的 angry-birds.html 后发现致命逻辑问题：

> **第一关打了第一下之后，就判定通过了第一关；点击「下一关」按钮后，会自动通关，一直到最后一关。**

这意味着游戏**核心循环完全失效**：
- 过关判定错误：未消灭所有猪即判定过关
- 关卡切换错误：点击下一关触发连锁自动通关

**后果**：游戏无法正常游玩，失去了作为"愤怒的小鸟"的意义。

**为什么代码审查没发现**：GLM5.2 的代码物理引擎实现非常专业（逆质量系统 invMass、基于 dt 的帧率无关物理、圆-圆/圆-矩/矩-矩三碰撞对、材质冲量阈值伤害模型、弹道预测线、粒子特效）——单看物理引擎代码质量是教科书级的（859 行，比 Fable5 的 663 行更复杂）。但**游戏循环逻辑（过关判定/关卡推进）存在致命缺陷**，这部分代码审查时被物理引擎的"专业感"掩盖了。

> 💡 这正是 Rubric 强调"必须实跑验证"的原因：代码的局部质量（物理引擎）不等于整体可用性（游戏循环）。GLM5.2 在这个题目上"局部优秀、整体失败"。

#### Fable5 愤怒小鸟：可正常游玩

测试人实跑反馈："**Fable5 的愤怒的小鸟好多了**"。Fable5 的版本可正常游玩，物理手感、过关逻辑、关卡切换均正常。代码层面：冲量法碰撞、材质区分（wood hp=70 / stone hp=160）、防抖动、2 个关卡、弹道预测线、粒子特效。

扣分项（未达满分原因）：
- request failed 频繁：D1-A 2 次，D1-B 高达 6 次（虽最终自主完成，但生成过程不稳定）
- 界面朴素：测试人评价"界面有点简陋"

#### 贪吃蛇：两者均可用

测试人实跑反馈："贪吃蛇目前看没什么大问题，两边都可以用，就是 GLM 多了个速度模式。"

两者 AI 均采用相同的三层决策策略（当前最优解共识）：BFS 寻路+安全验证 → 追尾巴保命 → floodFill 最大空间兜底。GLM5.2 额外有速度切换、暂停、WASD 支持。

GLM5.2 D1-B 扣分点：**没有直接输出为 HTML 文件，而是给了一堆源代码，二次 prompt 后才成功**（指令遵循瑕疵）。

#### 代码物理引擎特征对比（仅供参考，实跑结果优先）

| 检查项 | Fable5 小鸟 | GLM5.2 小鸟 |
|--------|:---------:|:---------:|
| 冲量法碰撞 | ✅ | ✅ |
| invMass 逆质量系统 | ❌ | ✅ |
| 基于 dt 时间步进 | ❌ | ✅ |
| 圆-圆/圆-矩/矩-矩 三碰撞对 | 部分 | ✅ 全覆盖 |
| 材质区分伤害 | ✅ | ✅ |
| 弹道预测线 | ✅ | ✅ |
| 粒子特效 | ✅ | ✅ |
| **游戏可正常通关** | ✅ | ❌ **致命** |

> ⚠️ 代码特征对比仅供参考。**实际评分以实跑结果为准**——GLM5.2 代码虽"看起来更专业"，但实跑不可玩。

### D7 · 知识广度与事实准确性（权重 13%）

#### 维度得分：Fable 5 = 4.0 / GLM-5.2 = 4.8

> 📌 **本维度 GLM-5.2 明确领先**，但原因特殊（见下）。

#### Fable 5 的 D7 困境

Fable 5 在 D7-B（抗幻觉测试）时**触发了 Claude Desktop 平台安全机制**：

> *Fable 5's safeguards flagged this message. Switched to Opus 4.8.*

平台**自动切换到 Opus 4.8 代为回答**。因此：
- D7-B 的优秀表现**归功于 Opus 4.8，不是 Fable 5**，不计入成绩。
- Fable 5 在 D7-B 的真实表现 = **触发 safeguard，无法独立评测**。
- Fable 5 的 D7 得分**仅基于 D7-A**（6/6 全对），数据不完整 → 给 4.0 而非更高。
- **建议**：用 Fable 5 的 API 直连（绕过 Desktop safeguard）重测 D7-B。

#### GLM-5.2 的 D7 优势（原生可测）

GLM-5.2 走 ccswitch → 官方 API，无平台 safeguard 干预，**完整完成了 D7-A + D7-B**：

- D7-A：6/6 全对（光速、马尔克斯、量子叠加/纠缠、皮肤陷阱、1929股灾、海拔沸点）
- D7-B：**原生完美识破 2 个陷阱**
  - "让-皮埃尔·杜布瓦"（不存在的作家）→ "这个不存在，我不会编造代表作"
  - "赫尔曼·冯·施瓦茨柴尔德"（名字编造）→ 纠正为 Karl Schwarzschild，**还额外指出卡尔的父亲确实叫 Hermann**！
  - 另纠正渥太华条约签署年份 1997 非 1998

> ⚠️ **公平性提示**：GLM5.2 的 D7 领先，部分原因是测试环境差异（GLM 可独立评测，Fable5 被 safeguard 拦截）。若 Fable5 能 API 直连重测 D7-B，分数可能上调。但就本次测试而言，GLM5.2 D7 数据更完整、抗幻觉能力经独立验证。

## 五、结论与选型建议

### 能力层面：Fable 5 略胜（4.56 vs 4.03）
