# D1 · 复杂代码生成与可运行性（权重 20%）

> **为什么这是第一维度**：一次性生成完整、能跑、好玩的程序，是对模型综合能力的最高考验——它要求模型同时具备逻辑、物理直觉、UI 设计、状态管理和工程化能力。这也是社区（Linux.do「愤怒的小鸟」测试）公认最有区分度的维度。
> **对标**：SWE-bench / HumanEval / 社区实战

---

## 测试任务 D1-A：生成「愤怒的小鸟」

**复制以下 Prompt 原样发给模型：**

```
请用纯 HTML+CSS+JavaScript（单个 HTML 文件，不依赖任何外部库和 CDN），实现一个「愤怒的小鸟」风格的弹弓射击小游戏。

要求：
1. 玩家用鼠标拖拽弹弓上的小鸟，松手后小鸟按抛物线飞出。
2. 必须有真实的物理效果：重力、弹性碰撞、小鸟与障碍物/猪的碰撞。
3. 画面上有多只绿色的猪和若干木块/石块障碍物，击中猪则得分，消灭所有猪即过关。
4. 至少 2 个关卡，关卡难度递增。
5. 有计分板、剩余小鸟数量显示、过关/失败提示。
6. 画面美观，元素有基本造型（不需要图片素材，用 Canvas 绘制圆形/矩形即可）。

请直接输出完整的、可以直接保存为 .html 文件在浏览器打开运行的代码，不要省略任何部分。
```

**执行**：把输出保存为 `angry_birds_<模型名>.html`，双击在浏览器打开实跑。

## 测试任务 D1-B：生成「贪吃蛇」+ AI 自动游玩

**复制以下 Prompt 原样发给模型：**

```
请用纯 HTML+JavaScript（单个文件，不依赖外部库），实现一个贪吃蛇游戏，并额外实现一个「AI 自动游玩」模式。

要求：
1. 经典贪吃蛇玩法：方向键控制，吃食物变长，撞墙或撞自己则 Game Over。
2. 必须有一个按钮可以切换「手动 / AI 自动」模式。
3. AI 模式下，蛇能自动寻路吃食物，且尽量不撞死（至少能合理地活比较久，不需要完美算法）。
4. 显示当前分数和历史最高分。
5. 画面用 Canvas 绘制，整洁美观。

请直接输出完整的、可保存为 .html 直接运行的代码。
```

**执行**：保存为 `snake_ai_<模型名>.html`，打开后**重点测试 AI 模式**能不能真的工作。

## 评分标准（Rubric，0–5 分制）

> 对每个任务分别打分，取平均值为 D1 维度得分。

| 分值 | 可运行性 | 游戏完整度 | 可玩性 / AI 智能 | 代码质量 | 视觉表现 |
|------|---------|-----------|----------------|---------|---------|
| **5** | 一次打开即完美运行，零报错 | 物理碰撞真实、计分/胜负/多关卡全部齐全，功能无缺失 | 手感流畅、难度合理；AI 真能自动寻路且存活久 | 结构清晰、有注释、命名规范、可扩展 | 元素有造型、配色舒适、有动效 |
| **4** | 运行正常，极小瑕疵 | 核心功能齐全，个别次要功能略有缺陷 | 可玩，AI 基本能动但偶有失误 | 良好，个别地方可优化 | 美观但较朴素 |
| **3** | 能跑但有可见 bug（穿模/卡顿/某个功能失效） | 基本能玩，但缺少 1-2 项要求功能 | 勉强能玩；AI 能动但很快死或逻辑差 | 能读懂但较乱 | 简陋但能辨认 |
| **2** | 能打开但主要功能不工作（如弹弓拖不动/蛇不动） | 严重缺失，只能看到静态画面 | 基本不可玩 | 结构混乱 | 难看/错位 |
| **1** | 报错白屏，或只有片段代码 | 几乎没有可玩内容 | — | 大量错误 | — |
| **0** | 拒绝生成 / 胡言乱语 / 无法保存 | — | — | — | — |

**特别记录项**（影响主观印象，附在评分后）：
- 🐛 Bug 清单：列出实跑发现的所有 bug
- 🔁 是否一次性生成（还是要追问才补全）
- 📦 代码行数
