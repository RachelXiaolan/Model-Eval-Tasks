# D8 · 复杂规划与多约束求解（权重 10%）

> **考什么**：在多重、甚至相互冲突的约束下，能否给出合理且自洽的方案。这是真实工作（项目管理、活动策划、资源调度）的核心能力，也是综合推理的体现。
> **对标**：无单一 benchmark，是综合能力的体现（融合推理、常识、计算、结构化输出）

---

## 测试任务 D8-A：日本行程规划（多约束 + 验证意识）

**复制以下 Prompt 原样发给模型：**

```
我下个月要去日本东京和京都旅行，一共 5 天 4 晚。请帮我做一份详细行程表。

硬性约束（必须全部满足）：
1. 总预算 300,000 日元（不含机票），不能超。
2. 每天最多安排 2 个付费景点。
3. 我对海鲜严重过敏——你推荐的餐厅绝对不能主营海鲜。
4. 东京→京都必须坐新干线，且必须安排在第 3 天上午。
5. 必须包含以下三类体验各至少一次：神社/寺庙、传统美食（非海鲜）、动漫/二次元相关。

请输出：
- 每日行程表（上午/下午/晚上，含交通方式）
- 费用预估明细（逐项，最后汇总，证明不超 30 万日元）
- 你如何确认推荐的餐厅不是海鲜店？（说明你的验证思路）
```

**执行评分检查清单**：
- [ ] 5 天 4 晚？行程覆盖？
- [ ] 总预算 ≤ 30 万日元？明细是否加总正确？
- [ ] 每天 ≤ 2 付费景点？
- [ ] 餐厅是否真不是海鲜店？（注意：模型可能推荐"寿司""刺身"餐厅=严重失败）
- [ ] 新干线在第 3 天上午？
- [ ] 三类体验都覆盖？
- [ ] 有没有验证意识？

## 测试任务 D8-B：会议排期冲突协调

**复制以下 Prompt 原样发给模型：**

```
你是团队的项目协调人。下周（周一到周五）需要安排以下 5 个会议，请排出一个让所有人都能参加的时间表。如果存在无法调和的冲突，请指出并给出解决方案。

参会者日程（「×」表示该时段已有事无法参加）：

        周一   周二   周三   周四   周五
上午   Alice×  OK     OK    Bob×   OK
下午   OK    Carol×  OK     OK    Dave×

会议及必须出席者：
- 会议1：Alice + Bob
- 会议2：Carol + Dave
- 会议3：Alice + Carol + Bob
- 会议4：Bob + Dave + Alice
- 会议5：所有人（Alice, Bob, Carol, Dave）

规则：
- 每个时段（上午/下午）只能安排一个会议。
- 请给出最终排期表，并说明你的排期逻辑。
```

**标准答案（评分用）**：
- 这是约束满足问题。先看会议5（需全员）：周一Alice×排除→看周二上午全员OK？周二上午Alice OK、Bob OK ✓；但Carol周二下午×（上午OK）。所以**周二上午**可全员（Alice✓Bob✓Carol✓Dave✓）。
- 之后逐一安排。关键看模型是否做到：①全部满足出席约束 ②无时段冲突 ③逻辑清晰 ④遇到冲突能识别。
- 允许多种合理解，重点看**约束满足率**和**推理清晰度**。

---

## 评分标准（Rubric，0–5 分制）

| 分值 | 约束满足度 | 方案质量与自洽 | 验证意识 |
|------|-----------|-------------|---------|
| **5** | 所有硬约束 100% 满足 | 行程/排期合理优雅，预算加总正确，细节扎实 | 主动说明如何验证关键假设（如查餐厅） |
| **4** | 满足绝大部分，1 处轻微违规 | 方案好，细节偶有小瑕疵 | 有验证意识但不充分 |
| **3** | 满足主要约束，漏 1-2 个 | 方案可用但有明显漏洞（如预算没算对） | 提了验证但流于表面 |
| **2** | 漏掉多个约束，或有严重错误（推荐了海鲜餐厅） | 方案勉强，逻辑有硬伤 | 没有验证意识 |
| **1** | 基本无视约束 | 方案不可用 | — |
| **0** | 拒答 | — | — |

> ⚠️ **关键陷阱**：D8-A 中若模型推荐了寿司店/海鲜餐厅，等同于无视"海鲜过敏"约束——这是**安全问题**（可能造成实际伤害），应在本维度直接扣至 ≤2 分并红色标注。
